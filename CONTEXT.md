# Custom WSL Ubuntu Distribution & Ecosystem Architecture

## 1. Goals & System Architecture
- **Distribution Base:** Ubuntu (Noble) customized for WSL2 with WSLg GUI support.
- **Development Focus:** Java 21 (via SDKMAN) and Angular (Node 20).
- **IDE & Tooling:**
    - IntelliJ IDEA Ultimate running natively inside WSL via Snap (`--classic`).
    - Antigravity running as a native Linux binary inside WSL (`~/.local/bin/antigravity`).
- **MCP Server Isolation:**
    - IntelliJ MCP server in WSL is configured to enforce port **64343** (overriding the default 64342) to prevent collisions with Windows IntelliJ.
    - Auto-configured on boot via `~/.bashrc` script patching `mcpServer.xml`.

## 2. Package Management & APT Repository
- **Local APT Repo Tool:** `reprepro`.
- **Hosting Strategy:** Static APT repository files generated by `reprepro` hosted on **GitHub Pages** (`https://<user>.github.io/<repo>`).
- **Custom Packages:** Custom tools (e.g., `k3d`, `kubectl`) wrapped into `.deb` packages via `dpkg-deb` and managed via `reprepro`.

## 3. Image Packaging & CI/CD Pipeline
- **Docker RootFS:** Built using a multi-stage `Dockerfile` that sets up systemd, packages, and the local APT source.
- **GitHub Actions Workflow:**
    1. Compiles `.deb` packages in `packages/`.
    2. Updates the `reprepro` static tree and deploys to GitHub Pages.
    3. Exports the Docker container filesystem into `.tar.gz` and `.wsl` distro installers.
    4. Publishes `.wsl` and `.tar.gz` bundles to GitHub Releases for `wsl --import`.
